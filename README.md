# 🧹 HygieneBot

HygieneBot is a completely serverless, automated AWS monitoring and cleanup application. It is designed to run asynchronously using EventBridge and SQS to identify unused, 'Zombie' cloud resources and orchestrate a Human-in-the-Loop cleanup pipeline entirely via Slack.

## 🏗️ Architecture

The infrastructure relies on the principle of least privilege and is fully provisioned using **Terraform**.

*   **Scanner Lambda**: Triggered automatically every Friday at Noon UTC via an **EventBridge** schedule. It utilizes Boto3 to detect:
    *   Unattached EBS Volumes
    *   Idle EC2 Instances (<1% CPU over 7 days)
    *   Untagged resources (orphaned instances)
    *   Old EBS Snapshots (>90 days retention)
*   **SQS Queue**: Because real-world cloud environments can have 10,000+ unused resources, passing them around synchronously isn't scalable. The Scanner Lambda packages individual items and offloads them to an SQS Queue.
*   **Slack Webhook & Interactivity**: Sends a summary report to Slack using Block Kit UI, presenting precise **Approve Cleanup** and **Deny** buttons.
*   **API Gateway & Deleter Lambda**: Provides the HTTPS interconnect for Slack's action button callbacks. It comprehensively evaluates `HMAC SHA-256` Slack signature verification using keys provisioned in **AWS Secrets Manager**. Upon valid approval, it asynchronously drains the SQS queue and executes the physical destruction commands against the AWS EC2 APIs.

## 🛠️ Deployment Instructions

### 1. Pre-requisites
*   An active AWS Account and configured AWS CLI.
*   A Slack Workspace where you can create a Custom App.

### 2. Configure Slack
1. Go to [api.slack.com/apps](https://api.slack.com/apps) and create **HygieneBot**.
2. Enable **Incoming Webhooks** and save the Webhook URL.
3. Copy the **Signing Secret** under Basic Information.

### 3. Deploy via Terraform
```bash
# Package the serverless functions
zip scanner_lambda.zip scanner_lambda.py
zip deleter_lambda.zip deleter_lambda.py

# Deploy the infrastructure
terraform init
terraform apply
```

### 4. Provide the Secrets
Terraform handles the IAM and scaffolding, but you must manually inject the Slack secrets into AWS Secrets Manager for absolute security:
```bash
aws secretsmanager create-secret --name hygienebot/slack --secret-string '{"webhook_url":"https://hook...", "signing_secret":"..."}' --region us-east-1
```

### 5. Finalize Interactivity
Under the **Interactivity & Shortcuts** tab in your Slack App configuration, paste the `api_gateway_endpoint` URL generated by Terraform into the **Request URL** box so the buttons work.

## 🚀 Testing
You can force-invoke the scanner directly via CLI to test the flow:
```bash
aws lambda invoke --function-name HygieneBotScanner --region us-east-1 /dev/stdout
```
